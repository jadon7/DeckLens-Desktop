# DeckLens Quality Rules

这是 DeckLens 转换的完整质量规则。任何主 Agent、子 Agent、单 Agent 路径都必须读取并执行本文件。不要只读取路由文件或简化流程文件。

## 最重要的交付门槛

| 场景 | 必须执行 | 不允许 |
| --- | --- | --- |
| 用户要求“拆分”“分层”“还原为可编辑 PPT” | 默认运行元素分层，并完成后处理验收 | 只返回 DeckLens 原始输出 |
| DeckLens 拆出大量碎片图层 | 按语义对象手动合并或替换为一个对象 | 把碎片图层原样交付 |
| 一个肉眼单一图片区域出现一堆控制框 | 从原图裁切完整区域，合并成一张位图，并删除所有碎片 | 保留多个相邻、重叠、嵌套图片碎片 |
| 程序分层拆分或合并不合理 | 基于原图语义手动拆分、合并、裁切或重建 | 被原始 mask 限制，明知不好还照单交付 |
| 页面里有独立矩形、圆形、卡片、按钮容器、分割线、图标 | 转换前列出 shape/icon 计划，转换后落实替换 | 等用户提醒后才补做 |
| 页面里有独立纯色背景、纯色区块、普通卡片 | 优先用 PPT 原生形状重建 | 直接沿用被拆分后的整块图片，或随手 SVG 化 |
| 页面背景、圆形、圆角矩形、卡片可由基础图形表达且是独立版式对象 | 默认加入 PPT shape 重建清单 | 只因为它不是 icon 就跳过判断，或把复杂图内部色块也强行拆出 |
| 页面里有卡片容器 | 卡片本体必须用圆角矩形/描边/阴影/渐变重建 | 因为卡片里有图片或文字就把整张卡片保留为位图 |
| 页面里有柱状图、条形图 | 用 PPT 矩形和文本重绘 | 把柱状图、条形图当作普通位图保留 |
| 页面里有饼图、环形图、折线图、面积图、雷达图或复杂图表 | 默认保留为位图，除非用户明确要求重绘 | 擅自 SVG 重绘导致风格和数据失真 |
| 页面里有背景渐变、底部色带渐变、按钮渐变、卡片渐变 | 还原渐变方向、主要色阶和覆盖范围 | 用单一纯色替代明显渐变 |
| 扁平 icon | 一律用 DeckLens 内置 icon 库检索和渲染，并删除源位图 | 用 emoji、Unicode 字符、普通文本、随手 SVG 或手绘几何蒙混过关 |
| 扁平柱状图和条形图 | 一律用 PPT 形状重新绘制 | 把柱状图、条形图作为图片交付 |
| 独立扁平矩形和圆形 | 优先用 PPT 原生形状重绘 | 保留独立基础图形图片层，或用 SVG 过度绘制 |
| 独立扁平背景 | 用 PPT 背景、矩形或渐变形状重建 | 把纯色/简单渐变背景当作整张位图保留 |
| 页面里有任何 icon 候选 | 必须运行 `decklens icons find` 和 `decklens icons render`；无法匹配时保留原位图或裁切原图并说明原因 | 不调用 icon 命令却声称完成图标替换，或自己手绘替代 |
| 新增 shape/SVG/icon | 对应源图片删除数量必须大于 0，或明确证明该对象原本没有源图片层 | shape 盖在旧图片上，最终仍能选中旧碎片 |
| OCR 文本回写到 PPT | 单行段落用自动宽度；同样式多行文本用固定宽度、自动高度 | 把同一段文字拆成多个互相漂移的文本框 |
| 准备交付前 | 运行最终 `inspect`，并按自检清单复核碎片、卡片、背景、icon、重复层 | 把结果交给用户肉眼兜底检查 |

如果页面有合规范围内的扁平候选（扁平 icon、扁平柱状图/条形图、独立扁平矩形/圆形/分割线或独立扁平背景容器），但最终没有任何符合规则的 shape/SVG/icon/vector 类对象，该页直接失败。不能把结果称为最终版本。

## 绘制范围边界

目标不是把所有能绘制 SVG 的东西都绘制成 SVG。目标是提高可编辑性，同时保持原图还原度。

优先级：

1. 标准 icon：优先 DeckLens 内置 icon 库渲染。
2. 卡片、背景、矩形、圆形、按钮容器、分割线：只用 PPT 原生 shape。
3. 柱状图、条形图：优先 PPT 矩形、线条和文本。
4. 复杂图片、截图、3D、照片、复杂插画、品牌 logo、非柱状/条形图图表：优先位图保留或完整裁切合并。
5. SVG 只用于 icon 库输出。非 icon 的矩形、圆形、卡片、按钮容器、分割线用 PPT 原生 shape；不要把箭头、手绘样式、复杂图形、照片局部、截图、logo、非柱状/条形图图表、复杂装饰强行 SVG 化。

禁止：

- 用 emoji、Unicode 字符、普通文本、字体符号假装 icon。
- 自己手绘扁平 icon 来替代 icon 库。
- 为了追求“矢量化数量”而绘制不该绘制的 SVG。
- 将照片、3D、复杂插画、截图、品牌 logo、复杂图表、复杂纹理拆成大量 SVG。
- 对原图中应作为一个复杂位图的区域进行局部 SVG 临摹。

## 扁平元素重绘边界

判断为扁平的条件：

- 主要由纯色、少量颜色、简单透明度或简单线性渐变构成。
- 没有照片细节、纸张/噪声纹理、3D 材质、复杂柔光、真实阴影或复杂插画笔触。
- 边界可以用矩形、圆形、分割线、图标库或柱状/条形图 primitives 近似表达。
- 它在原图中是独立语义对象，而不是复杂截图、照片、logo、3D、复杂插画或非柱状/条形图内部的一块局部色彩。
- 即使 DeckLens 分割结果有毛边，只要原图语义是扁平对象，也按扁平对象处理。

强制规则：

1. 但凡是扁平 icon，就一定要替换。必须用 `decklens icons find` + `decklens icons render`，至少尝试 2 个同义词或相近词。找不到标准图标时，不允许自己手绘，保留原位图或裁切原图并说明原因。
2. 但凡是扁平柱状图或条形图，就一定要重新绘制。用 PPT 矩形、线条和文本，尽量不用 SVG。
3. 饼图、环形图、折线图、面积图、雷达图和复杂图表默认不重绘，除非用户明确要求。需要保留为单张位图或完整截图区域，并说明“非柱状/条形图，按规则保留”。
4. 但凡是独立扁平矩形、圆形、椭圆、圆角矩形、胶囊、按钮容器、图标底座和分割线，就一定要用 PPT 原生 shape 重绘，不用 SVG。箭头、手绘线条、手绘标注、复杂装饰和非基础形状默认保持原图构造；如果箭头本身是标准 icon，只能用 icon 库替换。
5. 但凡是独立扁平背景，就一定要重绘。纯色背景用页面背景或全页矩形；简单渐变背景用 PPT 渐变或裁切为底层背景位图后再重建前景元素。

只有下面情况允许保留位图，并且必须逐项写入质量审计表：

- 元素含照片、3D、复杂插画、复杂纹理、复杂噪声、强拟物阴影或无法用少量矢量对象近似的视觉细节。
- 该对象不是独立扁平元素，而是复杂截图、照片、logo、非柱状/条形图或插画内部的一部分。
- 当前 PPT 原生形状或 icon 库无法表达关键视觉，并且保留位图比重绘明显更接近原图。

失败条件：

- 最终 PPTX 中仍有可解释为标准扁平 icon 的 picture，且没有记录 icon 库尝试失败和保留原因。
- 最终 PPTX 中仍有可解释为扁平柱状图或条形图的 picture。
- 最终 PPTX 中仍有可解释为独立扁平矩形、圆形、圆角矩形、按钮、分割线、图标底座的 picture。
- 最终 PPTX 中仍有可解释为独立纯色或简单渐变背景的 picture。
- 最终 PPTX 中有 emoji、Unicode 字符或普通文本充当 icon。
- 最终 PPTX 中有自己手绘的扁平 icon 替代 DeckLens icon 库。
- 最终 PPTX 中把非柱状/条形图、logo、截图、照片、3D 或复杂插画强行 SVG 化，且用户没有明确要求。
- 最终 PPTX 中把箭头、手绘样式、复杂装饰或非基础形状强行重绘成 SVG/shape，且它们不是标准 icon 库替换结果。
- 质量审计表里对合规范围内扁平候选只写“保留”但没有逐项原因。

## Agent 手动拆分和合并

DeckLens 的程序分层结果只是中间草稿，不是最终图层结构。必须以原图语义为准，而不是以 mask 数量、mask 边界或程序输出的 layer 数量为准。

必须手动合并：

- 同一张照片、人物、头发、服装、3D 物体、复杂插画、纹理图、截图缩略图，被拆成多个相邻、重叠或嵌套图片层。
- 多个碎片共享同一个视觉边界或外接矩形，选中时出现一堆控制框，但肉眼看起来就是一张图片。
- 项目预览图、案例截图、作品缩略图、黑底图片区域、人物照片或产品图里出现超过 2 个相邻/重叠 picture。

处理方式：从原始输入图裁切完整语义区域，作为一张位图插回 PPT，并删除或隐藏所有对应碎片层。不要尝试逐块拼碎片。

必须手动拆分或重建：

- 一个大图片层里包含扁平卡片、扁平 icon、扁平柱状图/条形图、独立扁平矩形/圆形、文本或背景。
- 程序没有分出来的纯色背景、卡片容器、圆形图标底、柱状图/条形图柱体、图例、分割线。
- 一个区域同时包含容器和复杂内容时，拆成“容器 shape + 内部复杂位图 + 文本/icon”，不要整块保留。

## 卡片容器规则

卡片是版式结构，不是普通图片。只要原图中某个区域表现为卡片容器，默认必须矢量化卡片本体。

识别为卡片的条件包括：

- 有明显矩形或圆角矩形边界。
- 有统一底色、半透明底色、描边、投影、毛玻璃、弱渐变或内边距。
- 多个同尺寸或同样式区块按网格、列表、横向卡组排列。
- 内部包含标题、说明、数字、图标、照片、截图或图表，但外部仍然有独立容器边界。

卡片后处理必须拆成两层：

1. 容器层：用 PPT 圆角矩形、填充、描边、阴影、透明度、渐变重建。
2. 内容层：文字用可编辑文本；图标用内置图标或 SVG/PNG；照片、截图、复杂插画可作为裁切位图保留。

如果卡片里有照片或复杂图像，只保留内部复杂图像为位图，不能因此把卡片背景、圆角、描边、投影也整体保留为位图。

卡片必须尽量还原原图样式：

- 圆角半径按原图视觉比例估算，不要统一套默认圆角。
- 尺寸、位置、圆角半径必须来自原图观察或测量，不能为了“好看”随意放大、缩小或改圆角。
- 填充色、透明度、描边颜色、描边粗细、阴影方向、阴影模糊、阴影透明度、渐变方向和渐变色阶都要单独判断。
- 同一页同组卡片要统一尺寸、圆角、描边、阴影和内边距；不同样式卡片不要强行统一。
- 卡片容器应位于内部文字、icon、截图下方；不要盖住内容。

每页验收前必须做卡片审计：原图肉眼可见卡片数量、最终被 shape 重建的卡片数量、圆角/填充/描边/阴影/渐变是否匹配、仍保留位图的卡片数量和原因。

## Icon 审计

只要原图出现图标、社交媒体标识、功能符号、状态符号、箭头、勾叉、用户、文件、屏幕、增长、目标等常见符号，就必须执行 icon 审计。

icon 审计必须包含：

- 每页 icon 候选数量、位置和语义。
- 对每个候选运行 `decklens icons find <语义名> --json`。第一次失败时至少尝试 2 个同义词或相近词。
- 对匹配成功的候选运行 `decklens icons render ... --format svg`，若 SVG 预览异常再渲染 PNG。
- 在 PPTX 中插入渲染后的 SVG/PNG，并删除对应源图片碎片。
- 插入 icon 时保持 1:1 比例。若原图候选框不是正方形，用 `min(width, height)` 作为 icon 尺寸，按原候选框中心点居中放置。
- 不允许用 emoji、Unicode 字符、字体符号、单个文字字符来替代 icon。
- 不允许跳过 DeckLens icon 库直接手绘常见 icon。
- 不允许在 icon 库未匹配时自己画一个“差不多”的 icon。找不到就保留原位图或裁切原图，并写入保留原因。
- 对品牌 logo、社交 logo、产品 logo，如果 icon 库无法匹配，不要臆造 SVG；保留原位图或裁切原 logo，并说明原因。

失败条件：

- 原图有 icon 候选，但没有任何 `decklens icons find` 调用。
- 原图有标准 icon 候选，但没有任何 `decklens icons render` 产物。
- 原图有扁平 icon 候选，但最终用手绘 SVG/PPT 几何替代。
- 最终 PPTX 里出现 emoji、Unicode 字符、字体符号、普通文本字符充当 icon。
- 最终 PPTX 中仍有可解释为 icon 的小 picture，而没有说明保留原因。

## 图表、背景和渐变

只重绘柱状图和条形图：

- 柱状图/条形图用矩形。
- 坐标轴、网格线、图例、数据标签用线条和文本。

数据值无法精确读出时，允许按视觉比例近似，但要保留原来的标签、颜色和图例顺序。

不重绘：

- 饼图、环形图、折线图、面积图、雷达图、复杂仪表盘、复杂数据可视化。
- 这些图表应保留为完整位图或截图区域，避免数据和视觉失真。

背景和渐变：

- 纯色背景、纯色区块、卡片容器优先用 PPT shape 重建。
- 背景渐变、底部 CTA 色带渐变、按钮渐变、卡片渐变必须还原方向、主要色阶、透明度和圆角。
- 不能等效重建渐变时，允许裁切原图对应区域作为底层位图保留，但文字、图标、线条等前景元素仍应保持可编辑。
- 不要把明显渐变简化成单一纯色。

## 图片碎片审计

每页 `inspect` 后必须统计：

- `picture` 总数。
- 小图片数量：宽和高都小于 `0.8in` 的 picture。
- 重叠图片组：多个 picture 的 bbox 高度重叠或互相嵌套，并且肉眼属于同一图像。
- 背景、照片、截图、纹理、图表、icon、卡片边框等语义分类。

触发强制清理：

- 单页小图片数量超过 6。
- 单页 picture 总数超过该页语义位图对象数量的 2 倍。
- 多个 picture 位于同一张照片、人物、3D、截图、纹理或插画范围内。
- 小 picture 看起来是图标、装饰点、边框碎片、阴影碎片、蒙版残片。

最终 `inspect` 的 picture 数必须能逐个解释为语义对象。不能解释的 picture 都要继续清理。

## 文本图层规则

OCR 文本回写到 PPT 时，文本框尺寸必须按文本语义定义：

- 单行段落、按钮文字、导航项、标签、数字指标：使用自动宽度。
- 同一段话被 OCR 成多行、或原图里本来就是多行说明文字：使用固定宽度、自动高度。
- 同一 style 的多行项目列表：每个列表项可以独立文本框，但宽度、字号、颜色、行高和 x 坐标要统一。
- 不同 style 的文字不要强行合并，例如标题和副标题、数字和单位、粗体和浅色说明。
- 同一段文字不要因为 OCR 分词被拆成多个互相漂移的文本框。
- 多行说明文字必须避免互相重叠。

## 每页质量审计表

每页必须返回：

| 项目 | 必填内容 |
| --- | --- |
| 图片层统计 | 原始 picture 数、最终 picture 数、原始小图片数、最终小图片数、删除/隐藏源图片层数 |
| 手动拆分/合并 | 程序分层不合理对象数、手动合并数、手动拆分数、手动重建数、未处理原因 |
| icon 替换 | icon 候选数、扁平 icon 数、find 调用数、render 产物数、已插入数、保留为位图数和原因、emoji/Unicode 替代数量 |
| 卡片重建 | 卡片候选数、shape 重建数、圆角/填充/描边/阴影/渐变匹配结论、保留位图数和原因、被删除的卡片背景/边框/阴影碎片数 |
| 背景和基础形状 | 独立背景/圆形/矩形/圆角矩形/分割线/按钮容器候选数、shape 重建数、尺寸/圆角/投影/描边/填充匹配结论、保留位图数和原因 |
| 图表重绘 | 柱状图/条形图候选数、已重绘数、其他图表保留位图数和原因 |
| 渐变处理 | 渐变候选数、PPT 渐变重建数、裁切位图保留数和原因 |
| 矢量对象计数 | 本页新增 shape/SVG/icon/vector 数量；如果合规范围内扁平候选数大于 0，此数量必须大于 0 |
| 重复叠加 | 新增对象对应删除源层数、重复叠加检查结果 |
| 图层顺序 | 每个新增对象是否放回原 z-order 语义位置，背景/卡片/图片/文字/icon 是否遮挡正确 |
| SVG 过度绘制 | 不该 SVG 化对象数量、已改回位图数量、仍异常原因 |

如果某页存在程序分层不合理对象，但手动合并数、手动拆分数、手动重建数都为 0，该页不通过。

## 最终验收

返回给用户前必须完成：

1. `inspect` 原始 DeckLens 输出。
2. 逐页建立替换清单，标明被合并的碎片层、复杂位图组和被删除的源位图层。
3. 执行矢量/icon/图表/卡片/背景后处理。
4. `inspect` 最终 PPTX。
