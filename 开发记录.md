# DeckLens 开发记录

本文记录当前已落地能力和关键技术取舍。历史 PRD、旧调参方案和废弃 LayerD/SAM 方案已清理，避免与当前实现冲突。

## 当前版本概览

- 产品名：DeckLens。
- 运行形态：Flask + Gunicorn 本地/容器服务。
- 主要输出：图片化页面重建为 PPTX；PDF 仅作为多页图片来源。
- 默认策略：标准还原优先，内存安全优先。
- 当前分支：`codex/unified-page-fonts`。

## 已落地能力

### 1. 图片化页面到 PPTX

- 支持 PNG/JPG 上传，也支持将 PDF 页面导入为图片。
- PDF 使用 PyMuPDF 渲染成页面图片，不解析 PDF 原始文本、矢量或图层结构。
- 多图片或多页 PDF 会合并导出为一套 PPTX。
- PPTX 尺寸按页面像素映射：`1px = 9525 EMU`。

### 2. OCR 与文字重建

- OCR 引擎：PaddleOCR 3.x。
- 每个 OCR 文本框输出为 PowerPoint 原生 textbox。
- 文本框使用 `SHAPE_TO_FIT_TEXT`、关闭自动换行、XML 级设置行距与段前段后。
- 文字颜色通过 K-Means/Otsu 从原图区域估算。
- 字号通过 Pillow 渲染测量和二分查找估算。
- 粗体通过字号与字符宽度启发式判断。

### 3. 字体匹配与页面级字体统一

- `TextBlock.font_name` 记录匹配字体。
- `match_fonts_for_blocks()` 对 OCR 区域调用本地 `font_matcher`。
- `normalize_page_fonts()` 在单页内统一字体：
  - 中文文本使用中文字体组。
  - 英文文本使用英文字体组。
  - 纯数字文本使用数字字体组。
  - 中英数混排优先使用中文字体组。
- 每页最多保留三种字体，减少输出 PPTX 视觉碎裂。

### 4. 文字擦除

- 默认后端：PyTorch LaMa，使用 OCR 文本框硬蒙版清理底图。
- 轻量回退后端：OpenCV `cv2.inpaint`，可通过 `DECKLENS_INPAINT_BACKEND=opencv` 显式启用。
- LaMa 默认按长边 1024 做推理，以控制耗时和内存。

### 5. 元素分层

- 标准还原：背景整图 + 可编辑文字。
- 元素分层：先生成预览，用户可合并/删除候选元素，再生成 PPTX。
- 默认本地分割：OpenCV 背景差异 + 连通域。
- SAM 保留为高级可选能力，需设置 `DECKLENS_ENABLE_SAM=1`。
- SAM 模型默认从 ViT-H 改为 ViT-B，并加入断点下载、完整性校验和失败 fallback。

### 6. AI 智能分层

- Qwen-Image-Layered 通过 fal.ai 调用。
- 需要用户设置 `FAL_KEY`。
- 输出 RGBA 图层后，会按 alpha 连通域拆成更小的独立 picture shape。
- fal.ai 不可用时会尝试 HuggingFace Space 后备路径；如果仍失败，则降级。

### 7. Web UI 与任务流

- `/api/convert` 创建任务。
- `/api/status/<task_id>` 查询进度。
- `/api/preview/<task_id>` 返回元素分层预览数据。
- `/api/confirm/<task_id>` 根据用户合并/删除决策生成 PPTX。
- `/api/download/<task_id>` 下载结果。
- `/healthz` 用于健康检查。

## 关键修复记录

### SAM checkpoint 不完整

问题：模型下载中断后，缓存里会留下损坏 `.pth`，再次加载时报 `checkpoint file is corrupted` 或 `IncompleteRead`。

处理：

- 下载到 `.part` 文件。
- 校验最小文件大小。
- 支持 Range 断点续传。
- 下载失败时保留 `.part` 并抛出明确错误。
- 默认不加载 SAM，避免无意触发大模型路径。

### 内存峰值过高

问题：本地测试出现 30GB 到 70GB 级内存压力。

根因：

- SAM automatic mask generation 在高分辨率页面上会生成大量全尺寸 mask。
- LaMa/Paddle/Torch 首次推理会保留框架缓存。
- 早期实现没有足够主动释放中间数组和预览数据。

处理：

- 默认关闭 SAM。
- 默认使用 PyTorch LaMa inpaint，OpenCV 作为失败回退。
- 限制本地处理分辨率。
- 任务完成后释放预览数据。
- 清理 `np.load` 文件句柄、临时目录和 Torch cache。

### 字体过于碎裂

问题：逐块字体匹配会让同一页出现过多字体，观感不统一。

处理：

- 新增 `text_font_category()` 和 `normalize_page_fonts()`。
- 同页字体按中文/英文/数字归一。
- 混排文本优先中文字体。

## 已废弃或清理的方向

- LayerD：对平面 PPT 设计图分层效果不稳定，相关 PRD 已删除。
- SAM 作为默认分层：内存峰值不可控，已改为显式 opt-in。
- LaMa 作为默认擦除：经素材对比后质量更好，接受较高首次加载成本；普通低配环境可用 OpenCV 回退。
- 多份旧 PRD 和旧测试调参文档：已删除，当前方案以 `README.md`、`方案文档.md`、`DEPLOYMENT.md` 和 `docs/ROADMAP.md` 为准。
