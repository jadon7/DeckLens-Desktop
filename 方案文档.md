# DeckLens 技术方案文档

## 项目目标

DeckLens 将图片化页面、AI 生成 PPT 图或截图重建为可继续编辑的 PPTX。当前目标不是恢复源 PPT 文件，而是在保留视觉观感的同时，让文字和主要视觉元素更容易二次编辑。PDF 仅作为页面图片导入来源：系统会先将 PDF 每页渲染为图片，再进入同一套处理流程。

## 当前处理流程

```text
上传图片或 PDF 页面导入
  -> PDF 渲染或图片读取
  -> 输入尺寸安全裁剪/降采样
  -> PaddleOCR 检测文字
  -> 字体匹配与页面级字体统一
  -> 颜色、字号、粗体启发式还原
  -> LaMa inpaint 擦除文字并修复底图
  -> 可选元素分层
  -> python-pptx 生成 PPTX
```

## 核心能力

| 环节 | 当前实现 | 说明 |
| --- | --- | --- |
| Web 服务 | Flask + Gunicorn | 异步线程处理任务，前端轮询状态 |
| 桌面壳 | Electron | macOS/Windows 桌面入口，复用 Flask Web UI |
| 输入 | PNG/JPG/PDF 页面导入 | PDF 通过 PyMuPDF 渲染为图片，不解析 PDF 原始结构 |
| OCR | PaddleOCR 3.x | 使用 PP-OCRv5 server 模型 |
| 字体 | 本地 `font_matcher` + 页面级归一化 | 每页最多中文、英文、数字三类字体 |
| 颜色 | K-Means + Otsu 回退 | 从 OCR 文本区域估算文字色 |
| 字号 | Pillow 二分测量 | 以文本框宽度为主约束 |
| 擦字 | PyTorch LaMa inpaint | 默认使用本地 LaMa，直接使用 OCR 文字框硬蒙版；OpenCV Telea 保留为回退 |
| 标准导出 | 背景图 + 可编辑文字框 | 最稳定的输出路径 |
| 元素分层 | OpenCV 兜底分割 + 交互预览 | SAM 默认关闭，避免内存峰值 |
| AI 智能分层 | fal.ai Qwen-Image-Layered | 需要 `FAL_KEY`，输出语义图层 |

## 字体统一规则

单页字体不再逐块无约束变化，而是按文本类型归一：

- 含中文：归入中文字体组。
- 不含中文但含英文：归入英文字体组。
- 纯数字：归入数字字体组。
- 中英数混排：优先归入中文字体组。

每组选择该页中出现次数最多的匹配字体；次数相同时，文本框面积更大的字体优先。最终每页最多三种字体。

## 内存安全策略

DeckLens 的模型和图像数组都可能占用大量内存，因此当前默认值偏保守：

- 默认使用 LaMa 擦字路径，输入长边按 `DECKLENS_LAMA_MAX_SIDE=1024` 限制。
- `DECKLENS_INPAINT_BACKEND=opencv` 可切回轻量 OpenCV Telea 路径，LaMa 失败时也会默认回退到 OpenCV。
- `DECKLENS_ENABLE_SAM` 默认关闭，避免 SAM automatic mask generation 在高分辨率页面上冲到几十 GB。
- `DECKLENS_MAX_PROCESS_MP=3.0` 和 `DECKLENS_MAX_PROCESS_SIDE=2200` 限制本地处理分辨率。
- 任务完成后释放预览数据，并执行 best-effort `gc` 与 Torch cache 清理。

## Electron 桌面方案

Electron 版本复用现有 Flask 应用，而不是重写前端或核心引擎。主进程负责：

1. 创建安全的 `BrowserWindow`，关闭 renderer 的 Node 集成，通过 preload 暴露受控安装/启动 API。
2. 将打包内的 Python 源码复制到 Electron `userData/backend`。
3. 检测系统 Python，首次使用时在 `userData/python-runtime/venv` 创建虚拟环境。
4. 安装 `requirements.txt` 中的 Python 依赖。
5. 用 Waitress 在随机本地端口启动 Flask：`python -m waitress --listen=127.0.0.1:<port> app:app`。
6. 健康检查通过后加载本地 DeckLens URL。

安装包只包含 Electron 壳、Python 源码、前端模板和字体匹配数据，不包含：

- `.venv` / `.venv312`
- PaddleOCR/PaddlePaddle/Torch 运行时依赖
- PaddleOCR 模型缓存
- SAM checkpoint
- LaMa 模型
- 用户上传文件或输出文件

这些资源在用户安装运行时或启用对应功能时再下载，从而保持 macOS/Windows 安装包相对较小。

## 分层策略

### 标准还原

默认路径。输出一张 LaMa 清理后的背景图和可编辑文字框，优先保证文字区域背景观感。

### 元素分层

本地轻量分割路径。生成预览页，用户可以选择、合并或删除候选元素，确认后生成 PPTX。当前默认使用 OpenCV 基于背景差异的连通域分割。SAM 仍保留，但必须显式设置 `DECKLENS_ENABLE_SAM=1`。

### AI 智能分层

当设置 `FAL_KEY` 时可用。系统将清理后的背景图上传到 fal.ai Qwen-Image-Layered，返回 RGBA 图层，再按 alpha 连通域拆成独立 PPTX picture shape。它质量更高，但依赖网络、外部 API 和用户自己的 fal.ai 额度。

## 已知边界

- 不能保证精确还原原始 PPT 的母版、动画、组合关系和全部矢量形状。
- 字体识别是视觉近似，不是原文件字体提取。
- 表格、列表、图表仍主要以文本框和背景图片形式还原。
- 元素分层对复杂纹理、渐变背景和半透明叠层仍可能过拆或漏拆。
- 当前任务状态存在内存中，因此生产部署应保持单 worker。
