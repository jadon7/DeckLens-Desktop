# Leadpages 风格参考

本文用于 DeckLens 后续 UI 迭代时复用。参考对象：<https://leadpages.com/>。这是保留的设计参考，不是实现规格；当前产品事实以根目录 `README.md` 和 `方案文档.md` 为准。

## 一句话判断

Leadpages 的页面不是“功能卡片堆叠”，而是“强主张 + 明确行动 + 产品演示 + 少量证据模块”。DeckLens 应借鉴这种信息密度：先让用户知道能完成什么，再给一个可操作工作台，最后用少量模块解释能力边界。

## 视觉关键词

- 深色沉浸背景，局部使用柔和渐变光线。
- 大字号、短句式标题，标题只表达一个核心价值。
- CTA 按钮颜色高对比，圆角克制，不做复杂装饰。
- 内容模块像“玻璃工作台”：半透明深色面板、细边框、轻微内高光、柔和阴影。
- 少用纯白大块卡片；如果出现浅色内容，应作为产品截图或演示画布，而不是普通信息容器。

## 信息结构

推荐顺序：

1. **主张**：用一句话说明用户最终得到什么。
2. **工作台**：把最主要操作前置，模式选择、上传、进度、结果都在同一区域完成。
3. **输出差异**：对比“原始状态”和“产品输出”，少讲技术，多讲用户体感。
4. **关键能力**：控制在 3 到 4 个模块内，避免重复表达。
5. **边界说明**：用短条说明不承诺什么，降低过度预期。

不推荐：

- 同一能力在流程卡、功能卡、说明条里反复出现。
- 大量并列卡片都使用相同权重。
- 在对外页面暴露模型、算法、库名等实现细节。

## 组件风格

### 顶部导航

- 品牌区左对齐，logo + 产品名 + 一句短定位。
- 导航项少而明确，3 到 5 个以内。
- 导航文字低对比，悬停再增强。

### Hero

- 标题应短，最好一屏内完成阅读。
- 支撑文案只解释结果，不解释过程。
- CTA 至少有一个强主按钮；次按钮要能触发真实工作流。

### 工作台

- 工作台是页面中心，不应像表单后台。
- 模式卡片应一行展示核心选择，卡片文字短，标题比说明更重要。
- 已选文件、进度、结果都应保留在同一面板内，减少视觉跳转。

### 进度状态

- 长任务不能只显示“处理中”。
- 推荐组合：阶段轨道 + 渐变进度条 + 当前步骤说明 + 下方阶段 feed。
- 秒数必须持续走，哪怕后端长时间卡在 OCR / 分割 / 生成阶段。

### 预览工作台

- 预览页应延续主页面风格：深色玻璃外壳 + 画布区域 + 左侧控制栏。
- 左侧列表要强调“选择、合并、删除”的操作，不要像调试数据表。
- 操作按钮尽量固定在同一区域，避免用户滚动寻找。

## DeckLens 套用原则

- 术语优先面向用户：标准还原、元素分层、批量成稿、可编辑文字。
- 对外少说技术实现：不写 SAM、Qwen、LaMa、PaddleOCR 等，除非是内部技术文档。
- 避免夸大：不说“完美还原”“100% 可编辑”“恢复源文件”。
- 强调结果：保留观感、文字可改、主要元素可整理、多页可合并。
