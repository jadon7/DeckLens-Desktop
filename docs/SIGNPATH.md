# SignPath Foundation Preparation

DeckLens is prepared for a SignPath Foundation application as an open-source
Windows desktop app.

## Repository Requirements

- Public GitHub repository.
- Open-source license in `LICENSE`.
- Reproducible Windows build workflow in GitHub Actions.
- Release artifacts generated by CI instead of by a local machine.
- No private test materials, credentials, generated release artifacts, or model
  weights committed to the repository.

## Current Windows Build

The `windows-build.yml` workflow builds the Windows NSIS installer and uploads
the unsigned artifacts:

- `DeckLens Setup *.exe`
- `latest.yml`
- `*.blockmap`

After SignPath approval, the signing step should be inserted between the
electron-builder output and the artifact upload/release publishing step.

## SignPath Application Notes

Use the public repository URL and point reviewers to:

- `LICENSE`
- `.github/workflows/windows-build.yml`
- `README.md`
- this document

The signing certificate and HSM-backed private key are managed by SignPath, not
by the repository or local developer machines.
