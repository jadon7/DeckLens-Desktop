# Release 0.2.19

## Context

On clean Windows computers, first-run runtime setup could fail with `Python 3.11 or 3.12 was not found`. The app attempted to install Python through `winget`, but Python discovery still depended on external install locations and PATH behavior.

## Decision

- Add an app-managed Windows Python 3.12.10 installation path under DeckLens user data.
- Download the Python installer from the Cloudflare update CDN first, with python.org as fallback.
- Verify the installer SHA-256 before running it.
- Install Python silently into the DeckLens runtime directory without changing user PATH.
- Keep `winget` as a fallback and make Python discovery check more Windows locations.
- Upload the Python installer to Cloudflare R2 with immutable cache.

## Consequences

Clean Windows devices no longer need a preinstalled Python runtime. DeckLens owns the Python used to create its local venv, which avoids PATH refresh and package-manager differences across user machines.
