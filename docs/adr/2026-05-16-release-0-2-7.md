# Release 0.2.7

## Context

The user manually refined the installed DeckLens Agent Skill under `/Users/jadon7/.agents/skills/decklens-convert`. The packaged skill should use that local version as the source of truth so future installs and skill updates distribute the same behavior.

## Decision

- Sync the local Agent Skill guidance into the repository skill source.
- Prefer element layering for image-like slide pages unless the user explicitly asks for another mode.
- Require a pre-conversion vectorization plan based on the original source image, not only post-segmentation masks.
- Add stricter icon handling rules: prefer standard icon sets, avoid emoji/text substitutes, keep icon style consistent, and report fallback behavior.
- Bundle the selected icon sources for Agent post-processing: `lucide-static`, `@tabler/icons`, `heroicons`, and `phosphor-icons`.
- Add a `decklens icons` CLI helper so Agents can resolve semantic icon names to local bundled assets without scanning packages or installing dependencies.
- Agents must not install npm icon packages or fetch icon sources during conversion.
- Publish 0.2.7 so the packaged skill and Cloudflare update feed distribute the updated instructions.

## Verification

- `decklens skills status` reports source version 0.2.7.
- Packaged macOS app reports packaged skill source version 0.2.7.
- Cloudflare update metadata points to version 0.2.7 after release.
