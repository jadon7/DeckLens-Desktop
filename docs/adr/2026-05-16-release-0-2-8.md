# Release 0.2.8

## Context

Agent-side DeckLens workflows were spending unnecessary work on two mechanical tasks:

- inspecting PPTX OpenXML structure after conversion,
- locating and converting bundled icons into insertable assets.

The visual decisions for vector redraw and icon replacement should still be made by the Agent, but the product CLI should own repeatable file inspection and icon rendering.

## Decision

Release `0.2.8` adds:

- `decklens inspect <deck.pptx> --json` for slide item order, media paths, text boxes, dimensions, and summaries.
- `decklens icons render <name> --format svg|png --output <file>` for deterministic icon assets from bundled libraries.
- updated Agent skill instructions that use the product CLI for inspection and icon rendering instead of manual PPTX unzip/npm package work.
