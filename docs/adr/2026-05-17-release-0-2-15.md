# Release 0.2.15

## Context

Windows runtime validation showed that `paddlepaddle 3.3.x` can import successfully but fail during OCR prediction with a Paddle oneDNN/PIR runtime error. This means the app can start, but image conversion can fail on Windows devices even after torch-backed optional packages are removed.

## Decision

- Pin the Windows runtime to `paddlepaddle==3.2.2`, while keeping non-Windows requirements unchanged.
- Generate Windows-specific requirements that replace any generic Paddle requirement with the pinned Windows version.
- On Windows startup, check the installed Paddle version through package metadata and migrate existing runtimes to `3.2.2` when needed.
- Make the Windows optional package cleanup state-based, so torch-backed packages are removed if present even when a previous cleanup marker exists.

## Consequences

Windows devices keep the same product algorithms and default flow, but run against a Paddle version verified to complete OCR prediction. Existing Windows installs are repaired in-place on the next launch after update.
