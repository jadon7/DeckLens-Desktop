# Release 0.2.20

## Context

Clean macOS devices can also fail first-run setup with `Python 3.11 or 3.12 was not found`. macOS no longer guarantees a suitable system Python, and Finder-launched apps do not inherit shell-managed Python paths reliably.

## Decision

- Add an app-managed macOS Python 3.12.13 runtime under DeckLens user data.
- Use Astral `python-build-standalone` install-only builds for arm64 and x86_64.
- Download from the Cloudflare update CDN first, with GitHub as fallback.
- Verify SHA-256 before extraction.
- Extract with the system `tar` command and use the managed interpreter to create the app venv.
- Keep existing system Python discovery as a fast path when a supported Python is already present.

## Consequences

Clean macOS devices no longer need a preinstalled Python 3.11/3.12. The runtime remains outside the app bundle to keep the installer smaller and is cached per user after first setup.
