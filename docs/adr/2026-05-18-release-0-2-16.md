# Release 0.2.16

## Context

On Windows, installing an app update through the updater can look like a full reinstall when the NSIS installer UI is shown. Separately, existing runtime repairs after an app update were presented with the same setup wording as first-time dependency installation, which made update behavior confusing.

## Decision

- Install downloaded updates silently on Windows by calling `quitAndInstall(true, true)`.
- Keep non-Windows update installation behavior unchanged.
- Detect installed Windows runtimes that need compatibility repair before backend startup.
- Show an automatic "updating runtime" progress state for runtime repair instead of the first-install setup copy.
- Keep first-time runtime installation behavior unchanged.

## Consequences

Windows users should no longer see the full installer during an app update. If DeckLens needs to repair the local runtime after an update, users see a short upgrade progress screen and then enter the app automatically.
