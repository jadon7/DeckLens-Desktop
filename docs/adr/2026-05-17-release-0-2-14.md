# Release 0.2.14

## Context

Windows users still saw `torch\lib\c10.dll` initialization failures and Paddle/PaddleX reinitialization errors after 0.2.13. Existing user settings could still request LaMa, and existing Windows virtual environments still contained torch-backed optional packages that PaddleX could encounter during initialization.

## Decision

- Disable torch by default on Windows at the engine boundary, so requesting LaMa falls back before importing torch.
- Force saved Windows inpaint settings back to the local mean backend.
- Generate filtered Windows requirements that omit torch, torchvision, ultralytics, simple-lama-inpainting, and segment-anything.
- On Windows startup, sanitize existing runtimes by uninstalling those optional torch-backed packages before launching the backend.
- Serialize PaddleOCR construction and prediction with a process-wide lock to avoid concurrent PDX initialization.

## Consequences

Windows uses the lightweight local inpainting and OpenCV segmentation paths by default. macOS keeps the higher-quality LaMa/FastSAM paths. Existing Windows installs are repaired in-place on the next launch after update.
