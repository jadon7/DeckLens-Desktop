# Release 0.2.13

## Context

Windows users reported native menu chrome, runtime DLL initialization errors from torch-backed paths, and Paddle/PDX reinitialization errors. The settings panel also still exposed manual Agent Skill installation even though skills should be synced automatically.

## Decision

- Remove the Agent Skill section from Settings and sync bundled skills silently on desktop startup.
- Hide the Windows application menu and use a white hidden title bar overlay so native controls match the app surface.
- Default Windows inpainting to the local mean backend and Windows element segmentation to the OpenCV fallback path to avoid torch DLL initialization failures.
- Keep Paddle OCR alive by default on Windows to avoid PDX reinitialization in the same backend process.
- Fall back from LaMa to local mean inpainting if LaMa cannot load.

## Consequences

Windows avoids the fragile torch-backed startup paths by default while macOS keeps the higher-quality LaMa/FastSAM defaults. Agent skills remain available without a manual settings action.
