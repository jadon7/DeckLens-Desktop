# DeckLens Roadmap

本文只记录当前仍有价值的待办。已废弃的 LayerD、默认 SAM 和旧 PRD 方案不再保留。

## P0

1. **Electron runtime UX**
   - 当前 Electron 壳会在首次启动时安装 Python runtime。
   - 需要增加更明确的下载进度、失败重试、Python 缺失引导和依赖安装校验。

2. **任务持久化**
   - 当前任务状态存在进程内存中，只能安全使用单 worker。
   - 上线前应迁移到 Redis/Celery、数据库队列或其他后台任务系统。

3. **上传与输出清理**
   - `uploads/`、`outputs/`、`preview/` 需要自动过期删除。
   - 公共部署前需要明确文件保留时间。

4. **错误展示**
   - 当前后端错误会进入 task error，但前端仍可更具体展示：OCR 失败、Qwen 失败、文件过大、下载失败。

5. **标准测试集**
   - 建立小型固定样例，覆盖中文、英文、数字、PDF、多页、元素分层、Qwen fallback。

## P1

1. **段落合并**
   - 当前每个 OCR bbox 独立成文本框。
   - 可按行距、水平对齐和语义邻近合并为段落级文本框。

2. **对齐方式检测**
   - 根据文本块边界判断左对齐、居中、右对齐。
   - 对列表、标题、副标题输出更自然。

3. **表格识别**
   - 图片中的表格当前仍是背景 + 文本框。
   - 可评估 PaddleOCR table structure 或其他表格结构模型。

4. **列表结构识别**
   - 识别序号、项目符号和缩进，输出原生 PowerPoint bullet list。

5. **预览交互完善**
   - 空白取消选择。
   - Escape 取消选择。
   - Delete/Backspace 删除选中。
   - Ctrl+A 全选。
   - Ctrl+G 合并选中。
   - Ctrl+Z/Ctrl+Y 撤销重做。

## P2

1. **图形/箭头/图标还原为原生 shape**
   - 当前主要以 picture shape 输出。
   - 可逐步识别矩形、圆形、线条、箭头等基础形状。

2. **更细的字体策略**
   - 当前每页最多中文、英文、数字三类字体。
   - 后续可在保持统一的前提下识别标题/正文风格。

3. **Qwen 成本与账号配置**
   - 支持用户在界面中配置自己的 fal.ai API key。
   - 后端不长期保存 key，或加密保存到用户级配置。

4. **部署扩展**
   - 多 worker / 多实例。
   - 对象存储。
   - 任务队列。
   - 用户级鉴权与配额。

## 当前不做

- 默认启用本地 SAM。
- 默认启用 LaMa。
- 本地部署 Qwen-Image-Layered 20B。
- 承诺 100% 还原源 PPT。
